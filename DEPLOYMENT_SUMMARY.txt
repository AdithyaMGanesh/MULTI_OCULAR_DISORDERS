## 📋 API Endpoints

### 1. Health Check
```
GET /health
```
Returns system status and model availability

**Response:**
```json
{
  "status": "healthy",
  "specialist_model_loaded": true/false,
  "generalist_model_loaded": true/false,
  "system_mode": "dual|hybrid|demo"
}
```

### 2. Model Information
```
GET /model-info
```
Returns detailed model information

**Response:**
```json
{
  "system": "Two-Lane Highway Dual-Model",
  "lane_1_specialist": {
    "name": "DR Detection Model",
    "architecture": "Fusion VGG16+ResNet50+DenseNet121",
    "input_shape": [224, 224, 3],
    "num_classes": 5,
    "classes": {...},
    "loaded": true
  },
  "lane_2_generalist": {...}
}
```

### 3. Dual Prediction
```
POST /predict-dual
Content-Type: multipart/form-data
```
Uploads image and returns combined dual-model predictions

**Request:**
```
file: <image file>
```

**Response:**
```json
{
  "status": "success",
  "mode": "demo|production",

    "disease": "Diabetic Retinopathy",
```

## 🚀 Quick Start

### Prerequisites
- Python 3.8+
- Node.js 14+
- Internet connection (for npm packages)

### 60-Second Setup

**Terminal 1 - Backend:**
```bash
# Navigate to project
cd e:\DOWNLOADS\multi-ocular

# Start backend
python -m uvicorn src.api_dual_model:app --host 0.0.0.0 --port 8001
```

**Terminal 2 - Frontend:**
```bash
# Navigate to frontend
cd e:\DOWNLOADS\multi-ocular\frontend

# Install dependencies (first time only)
npm install

# Start frontend
npm start
```

**Access System:**
- Frontend UI: http://localhost:3000
- Backend API: http://localhost:8001
- API Docs: http://localhost:8001/docs

## 📊 Workflow Example

### User Flow
1. **Upload Image** → Browse or drag-drop retinal image
2. **Analyze** → Click "Analyze Image (Dual Model)"
3. **Processing** → Backend processes through both lanes:
   - Lane 1: Resizes to 224×224, runs DR model
   - Lane 2: Resizes to 128×128, runs Glaucoma/Cataract model
4. **Display Results** → Shows:
   - Lane 1 diagnosis with severity and probabilities
   - Lane 2 diagnosis with severity and probabilities
   - Combined risk assessment
   - Clinical recommendations

## 🐛 Troubleshooting

### Backend Won't Start
```
Error: Port 8001 already in use
→ Solution: netstat -ano | findstr :8001 and kill process
```

### Connection Refused
```
Error: Cannot connect to http://localhost:8001
→ Solution: Ensure backend is running in first terminal
```

### Frontend Won't Load
```
Error: CORS error or blank page
→ Solution: Check backend is running
→ Verify frontend API_BASE_URL points to http://localhost:8001
```

### Models Not Loading
```
Error: Running in DEMO MODE
→ Solution: Place models in root directory
→ Filenames must be exact:
  - fusion_dr_model.h5
  - custom_lightweight_model_v2.h5
```

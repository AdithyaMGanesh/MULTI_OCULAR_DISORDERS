🏗️ THE TWO-LANE HIGHWAY ARCHITECTURE
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

BEFORE (Single-Lane):
┌────────────────────────────────────────────────────────────────┐
│ Image → Resize (224×224) → DR Model → Result                 │
└────────────────────────────────────────────────────────────────┘
   Output: DR diagnosis ONLY

AFTER (Two-Lane Highway):
┌────────────────────────────────────────────────────────────────┐
│                      Retinal Image                             │
└────────────────┬────────────────────────────────────────────────┘
                 │
         ┌───────┴────────┐
         │                │
    LANE 1 ↓          LANE 2 ↓
    (224×224)        (128×128)
         │                │
    DR Model         G/C Model
         │                │
   5 Classes         3 Classes
         │                │
    DR Diagnosis     G/C Diagnosis
         │                │
         └───────┬────────┘
                 │
         Merge + Combine Risk
                 │
         Combined JSON Response
    (Both Diagnoses + Risk Level)

🎯 WORKFLOW
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Step 1: User Uploads Image
   ↓
Step 2: Backend receives image
   ↓
Step 3: Run parallel preprocessing
   ├─ Lane 1: Resize to 224×224
   └─ Lane 2: Resize to 128×128
   ↓
Step 4: Run parallel inference
   ├─ Lane 1: DR Model (Specialist)
   └─ Lane 2: Glaucoma/Cataract Model (Generalist)
   ↓
Step 5: Merge results & calculate combined risk
   ↓
Step 6: Return JSON with both diagnoses
   ↓
Step 7: Frontend displays results side-by-side
   ├─ Lane 1 diagnosis (Red/Amber/Green)
   ├─ Lane 2 diagnosis (Red/Amber/Green)
   ├─ Probability charts for both
   ├─ Combined risk assessment
   └─ Clinical recommendations
